## 🔍 What is Spatial Domain & Spectral Domain?

### ❓ Fundamental Question

How do we process facial landmarks in **two different mathematical spaces** to capture both local and global features?

### 📋 Unified Algorithm Configuration

| Property                 | Value                                                 |
| ------------------------ | ----------------------------------------------------- |
| **Algorithm Name** | Spatial & Spectral Graph Neural Network (SSGNN)       |
| **Input Format**   | Facial landmark coordinates (CSV data)                |

## 1️⃣ Algorithm Pseudocode

### 🔄 Complete Pipeline

```
╔════════════════════════════════════════════════════════════╗
║            SSGNN PROCESSING PIPELINE (v1.0)                ║
╚════════════════════════════════════════════════════════════╝

%START%

  ┌─ PHASE 1: GRAPH CONSTRUCTION
  │
  ├─ Calculate Euclidean distances between all landmark nodes
  │
  ├─ Define Adjacency Matrix (A) by connecting each node to 
  │  its K-nearest neighbors
  │
  ├─ Define Degree Matrix (D) by summing the connections 
  │  for each node
  │
  └─ Compute Laplacian Matrix: L = D - A


  ┌─ PHASE 2: PARALLEL FEATURE EXTRACTION
  │
  ├─── SPECTRAL DOMAIN PATH ─────────────────────────────────
  │    ├─ Perform Eigen-decomposition on L to find basis V 
  │    │  and eigenvalues Lambda
  │    │
  │    ├─ Transform spatial data to spectral domain using 
  │    │  Graph Fourier Transform (V^T * x)
  │    │
  │    ├─ Apply spectral convolution filters
  │    │
  │    └─ Compute probability y_spe using Softmax
  │
  └─── SPATIAL DOMAIN PATH ──────────────────────────────────
       ├─ Aggregate features from neighboring nodes using 
       │  spatial convolution weights
       │
       ├─ Apply ReLU activation and Average Pooling to capture 
       │  muscle synergy patterns
       │
       └─ Compute probability y_spa using Softmax


  ┌─ PHASE 3: DECISION FUSION
  │
  └─ Combine spatial and spectral predictions:
     model_result = fusion(y_spe, y_spa)

%END%
```

## 2️⃣ Code Implementation

### 📝 Python Implementation (NumPy + SciPy)

```python
import numpy as np
from scipy.linalg import eigh


def compute_ssgnn_logic(landmarks, k_neighbors=8):
    """
    Implements fundamental logic of SSGNN based on Formulas 1-4.
  
    Parameters:
    -----------
    landmarks : np.ndarray
        Shape (n, 2) - n facial landmarks with (x, y) coordinates
    k_neighbors : int
        Number of nearest neighbors for graph connectivity (default: 8)
  
    Returns:
    --------
    A : np.ndarray
        Adjacency matrix (n × n)
    D : np.ndarray
        Degree matrix (n × n)
    L : np.ndarray
        Laplacian matrix (n × n)
    eigenvalues : np.ndarray
        Eigenvalues of L
    V : np.ndarray
        Eigenvectors of L (Graph Fourier basis)
    """
  
    n = len(landmarks)
  
    # ═══════════════════════════════════════════════════════════
    # STEP 1: Graph Construction on Spatial Data
    # ═══════════════════════════════════════════════════════════
  
    # Calculate Euclidean distances between all landmark pairs
    dist_matrix = np.sqrt(
        np.sum((landmarks[:, np.newaxis] - landmarks[None, :])**2, axis=2)
    )
  
    # ═══════════════════════════════════════════════════════════
    # STEP 2: Build Adjacency Matrix A (Formula 2)
    # K-nearest neighbors connectivity
    # ═══════════════════════════════════════════════════════════
  
    A = np.zeros((n, n))
  
    for i in range(n):
        # Find indices of k-closest neighbors (excluding self)
        nearest = np.argsort(dist_matrix[i])[1:k_neighbors+1]
        A[i, nearest] = 1
  
    # ═══════════════════════════════════════════════════════════
    # STEP 3: Calculate Degree Matrix D (Formula 3)
    # Count connections for each node
    # ═══════════════════════════════════════════════════════════
  
    D = np.diag(np.sum(A, axis=1))
  
    # ═══════════════════════════════════════════════════════════
    # STEP 4: Compute Laplacian Matrix L (Formula 4)
    # Graph topology representation
    # ═══════════════════════════════════════════════════════════
  
    L = D - A
  
    # ═══════════════════════════════════════════════════════════
    # STEP 5: Spectral Domain Conversion
    # Eigen-decomposition: L = V * Lambda * V.T (Formula 5)
    # ═══════════════════════════════════════════════════════════
  
    eigenvalues, V = eigh(L)  # For symmetric matrices
  
    # V is the basis for Graph Fourier Transform
    Lambda = np.diag(eigenvalues)
  
    # ═══════════════════════════════════════════════════════════
    # STEP 6: Simulated Decision Fusion
    # (For demonstration of Formula 14)
    # ═══════════════════════════════════════════════════════════
  
    y_spe = np.array([0.1, 0.8, 0.1])  # Spectral domain output
    y_spa = np.array([0.2, 0.7, 0.1])  # Spatial domain output
  
    # Final Classification (Formula 14)
    final_emotion = np.argmax([y_spe, y_spa])
  
    return A, D, L, eigenvalues, V


# ══════════════════════════════════════════════════════════════
# USAGE EXAMPLE
# ══════════════════════════════════════════════════════════════

# Define 4 facial landmark coordinates (m × n) where m = 4
# N1 = (0, 1)  - Top landmark
# N2 = (1, 0)  - Right landmark
# N3 = (0, 0)  - Bottom landmark
# N4 = (0, 0)  - Center landmark

landmarks = np.array([
    [0, 1],
    [1, 0],
    [0, 0],
    [0, 0]
])

# Run SSGNN core logic
A, D, L, eig_val, eig_vec = compute_ssgnn_logic(landmarks, k_neighbors=2)

print("Adjacency Matrix A:\n", A)
print("Degree Matrix D:\n", D)
print("Laplacian Matrix L:\n", L)
print("Eigenvalues:", eig_val)
```

### 2.1 Mathematical Foundation: Eigenvectors & Eigenvalues

**What are Eigenvectors (v) and Eigenvalues (λ)?**

### 📐 The Eigen-Equation

The fundamental relationship in linear algebra is expressed as:

$$
Av = \lambda v
$$

### 🔑 Component Definitions

| Symbol       | Name        | Meaning                                                       |
| ------------ | ----------- | ------------------------------------------------------------- |
| **A**  | Matrix      | Square matrix representing a linear transformation            |
| **v**  | Eigenvector | Non-zero vector that only changes in magnitude, not direction |
| **λ** | Eigenvalue  | Scalar that represents the scaling factor                     |

### 💡 Interpretation in SSGNN Context

- **Laplacian Matrix (A = L):** Represents graph topology
- **Eigenvalues (λ):** Represent frequency components of the graph
- **Eigenvectors (v):** Basis vectors for spectral domain transformation
- **Application:** Graph Fourier Transform uses eigenvectors as basis

## 3. Architecture Branches

### 3.1 SPAGNN (Spatial-Domain Graph Neural Network)

**Focus:** Local spatial relationships and neighboring node dependencies

- Processes spatial domain directly
- Captures local muscle activation patterns
- Uses neighborhood aggregation
- Applies ReLU and pooling operations
- Outputs spatial probability distribution

### 3.2 SPEGNN (Spectral-Domain Graph Neural Network)

**Focus:** Global frequency components and spectral characteristics

- Transforms to frequency domain
- Captures global facial features
- Uses eigenvalue-based filtering
- Applies spectral convolution
- Outputs spectral probability distribution

---

## 4. Optimization Considerations

### 4.1 Why use SPAGNN for formulas 9, 10, 11, and 12?

**Formula Coverage:**

- **Formula 9:** Spatial Feature Aggregation
- **Formula 10:** Normalized Spatial Aggregation
- **Formula 11:** ReLU Activation (Spatial)
- **Formula 12:** Average Pooling

**Rationale:** SPAGNN formulas are designed to extract local spatial patterns directly from landmark coordinates without frequency transformation.

#### Q2: Why use softmax (formula 14) for SPAGNN?

**Purpose:** Convert pooled spatial features into probability distribution

**Answer:** Softmax normalizes the aggregated spatial features into a probability distribution over emotion classes.

#### Q3: Why use average pooling (formula 13) for SPAGNN?

**Purpose:** Aggregate node-level features into single representation

**Answer:** Average pooling captures overall muscle activation patterns and facial synergy information.

#### Q4: Why use SPEGNN for formulas 5, 6, and 7?

**Formula Coverage:**

- **Formula 5:** Eigendecomposition
- **Formula 6:** Graph Fourier Transform
- **Formula 7:** Spectral Convolution

**Rationale:** These formulas specifically operate in frequency domain and require spectral analysis.

## 5️⃣ Prediction Strategy

### ❓ Probability & Fusion Questions

#### Q1: Why use softmax (formula 8) for SPEGNN?

**Answer:** Softmax normalizes spectral convolution outputs into probability distribution, matching SPAGNN's output format.

#### Q2: Why use SPEGNN for formulas 5, 6, and 7?

**Answer:** These formulas are inherently spectral:

- **Formula 5** decomposes into frequency basis
- **Formula 6** transforms to frequency domain
- **Formula 7** applies frequency-domain filters

#### Q3: Why combine softmax with average pooling (formulas 13, 14)?

**Answer:**

- **Formula 13:** Spatial domain pooling → aggregated features
- **Formula 14:** Decision fusion → combines spatial + spectral predictions

---

## 6️⃣ Combined Configuration Strategy

### ❓ Integration Design Questions

#### Q1: Why NOT use softmax (formula 8) for SPAGNN and formula 14 for SPEGNN?

**Answer:** Domain-specific optimization

- **SPAGNN** uses formula 13 for spatial pooling, then formula 14 for final softmax
- **SPEGNN** uses formula 8 directly after spectral convolution
- Separating these prevents double-softmax effects

#### Q2: Why NOT use average pooling (formula 13) for SPEGNN?

**Answer:** Incompatible operations

- **Spectral domain** data has already been Fourier-transformed
- Average pooling operates on spatial coordinates
- Pooling after spectral convolution would lose frequency information
